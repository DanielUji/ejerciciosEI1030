Reflexión sobre el post

"7 RAZONES PARA MODELAR EL SOFTWARE ANTES DE PROGRAMAR"

En este punto he seleccionado el artículo “7 RAZONES PARA MODELAR EL SOFTWARE ANTES DE PROGRAMAR” ya que me parece muy interesante para comentar a lo largo de este apartado, ya que después de leerlo me ha parecido apropiado comentarlo. A lo largo del artículo expone distintos puntos por los cuales hay que tener en cuenta hacer un buen diseño previo antes de ponerse manos a la obra con el programa. En uno de ellos expone que el diseño previo ayuda significativamente para la producción en equipo, ya que partiendo de un modelo permite una mejor colaboración, y sobretodo permite reducir los defectos que puedan producirse en el código.
Otro punto importante es que después de hacer un diseño previo es posible explorar otras alternativas para hacer más eficiente el diseño y la implementación del mismo, manteniendo así un diseño más simplificado y completo que sin un diseño previo y sobretodo permite capturar eficientemente el conocimiento del sistema que se está desarrollando. Gracias a estos modelos se facilita el reutilizar de softwares para proyectos futuros para proyectos que requieren las mismas especificaciones.
Finalmente el autor del artículo que emplear diseños de este tipo resulta indispensable para que los clientes tengan un feedback del programa/proyecto por el que están pagando y así ver una evolución mientras el proyecto se lleva a término.
En lo que puedo considerar este articulo me parece muy interesante de leer, ya que te da una visión cara a un futuro profesional de lo que es este mundo que es el desarrollo de software se trata, no es simplemente desarrollar un programa, sino de esquematizar, organizar y diseñar previamente como primeros pasos y una vez asentadas estas bases empezar a desarrollarlo bien. Y respecto a los clientes es indispensable mostrarles que es lo que se les está ofreciendo a partir de su idea y así demostrar lo comprometidos que estamos en solventar su caso.

CU04 - Crear fiesta con eventos
Click para ampliar
Descripción del escenario

En este caso de uso el usuario una vez autentificado tiene la opción de de crear una fiesta, donde deberá introducir los datos de la misma y una vez creada esta, crear los distintos eventos a los que van a asistir los distintos invitados. No olvidar que el usuario debe añadir los datos del distinatario y a su vez gestionar los ditintos roles que van a desempeñar los invitados.

Listado de acciones de la especificacion del caso de uso representado

  1. El sistema pide al usuario sus datos personales.
  2. El usuario introduce los datos y se identifica como “Responsable”.
  3. El responsable decide crear una fiesta.
  4. El sistema le solicita al responsable introducir los datos de la fiesta.
  5. El responsable introduce todos los datos en los campos correspondientes.
  6. Se crea la fiesta.
  7. El responsable desea introducir el “Destinatario”.
  8. El sistema solicita los datos del destinatario de la fiesta.
  9. El responsable introduce todos los datos del destinatario.
  10. El sistema guarda todos los datos del destinatario.
  11. El responsable desea crear un evento para la fiesta.
  12. El sistema le solicita al responsable que introduzca datos del evento.
  13. El responsable introduce los datos del evento.
  14. Se crea el evento asociado a la fiesta.
  15. El responsable gestiona los invitados del evento.
  16. El sistema devuelve listado con todos los invitados.
  17. El responsable asigna roles a cada invitado.
  18. El sistema guarda los datos de cada rol de cada invitado.
  19. El usuario cierra la sesión.
  20. El sistema cierra la sesión de usuario.
